Module 2: Network Security

A 6-Unit Module that Started on June 2022 and was Covered in 6 Weeks by Beran Necat

Summary

In this Module, we looked at the impact of cyber security on network systems and designs by focusing on network vulnerability assessments.

We also discussed the techniques that can be used to scan, evaluate and in some case mitigate network vulnerabilities.

Skills Acquired during this Module

  1. Ability to identify and analyse security threat and vulnerabilities in network systems.

  2. Ability to gather and synthesise information from multiple sources (including internet security alerts and warning sites) so as to aid in the systematic analysis of security breaches and issues.

  3. Gained knowledge on how to create and use security vulnerability and assessment tools

  4. Gained Knowledge of the legal, social, ethical and professional issues faced by information security and risk professionals: Data privacy, etc.

  5. Ability to determine appropriate methodologies, tools and techniques to manage and/or solve security threats and vulnerabilities: Use of Threat methodologies like: CyberKill Chain model, use of penetration testing tools, Attack trees, OWASP, STRIDE, DREAD, etc.

  6. Awareness of the various protocols and architectures used by various network systems.

  7. Knowledge of the fundamentals of vulnerability assessment in various systems and the fundamentals of the provision of security in information networks.

    Tasks and Assignments Done throughout this Module

    Below is a breakdown of the activities done weekly during the 6-Unit Module.

    1. Week 1: Unit 1 - History of Network Security, Vulnerabilities and Approaches

      2. After going through a Lecture cast, books and a having a lesson with the Lecturer, I gained knowledge of the history of network security and vulnerabilities and related it to the modern world. We also discussed some security best practices and reviewed the Cyber Kill Chain model.

      We also got to review the origins of a number of common attacks (like CTSS first recorded vulnerability (OSVDB) in 1965, release of Melissa virus, the first mass emailer in 1999, Cyberterrorism attacks begin (Estonia) in 2007): And recommended best practice approaches that could be used to mitigate the attacks.

      A Collaborative Learning Discussion 1 was held, whereby we discussed "Digitalisation - What are the security implications of the digital economy?".

      3. We looked at Cyber Kill Chain model proposed by Lockheed Martin and applied it to a modern case study - the solar winds exploit.

      4. We had a Lecture cast and book reading that focused on general vulnerability assessment approaches that can be used to create a basic audit document.

    5. Week 5: Unit 5 - Logging, Forensics and Future Trends

      6. With the use of a Lecture cast, books and a lecture, we looked at the differences between logging on Windows and Linux systems and also reviewed key tools used in logging.

      We also reviewed the types of investigation that can be carried out using common forensic tools and the data that can be extracted.

      During the lecture, we reviewed an Assessment Reporting Template from PurpleSec. I learned how to apply the methods, tools and findings so as to create an assessment document.

    6. Week 6: Unit 6 - The Great Debate: The Future of the Internet

      7. It focused on the future of the Internet: How a number of alternative next generation internet solutions will operate and how these solutions address key challenges.

      In regards to the future of the internet, we reviewed:

      1. Future of the internet is based on content centric networking (CCN and/or NDN or COAST).
      2. future of the internet is based on peer-to-peer overlay-based networking (BitTorrent, TOR, Freenet, KAD).
      3. Future of the internet is based on the MobilityFirst architecture.
      4. Future of the internet is based on the adoption of IPv6 and the rollout of the associated security measures (DNSSEC, HTTP/3, IPsec, etc.)
